# AIMM ↔ WB Decision Log

## Objetivo
Registrar decisões metodológicas do mapeamento AIMM↔WB com rastreabilidade e versionamento.

## Campos padronizados por decisão
- Data
- Decision ID
- Categoria AIMM
- ID AIMM
- Proposta
- Decisão (Aprovada/Rejeitada/Revisar)
- Racional técnico
- Evidência (URL/endpoint/documento)
- Impacto na calculadora
- Responsável
- Próxima revisão

## Regras de governança
1. Nenhum mapeamento entra como “Aprovado” sem evidência oficial.
2. Toda decisão deve citar endpoint ou documento.
3. Mudanças em mapeamentos aprovados exigem novo Decision ID.
4. Revisão periódica mínima: trimestral.
